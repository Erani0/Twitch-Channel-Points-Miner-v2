StreamLayer v2 — Was bauen wir?

Das ist ein großes Vorhaben mit 4 unabhängigen Subsystemen. Wähle die Prioritätsreihenfolge — was zuerst?

1

🎬 Szenen-System

Start · Pause · Spiel · Ende — live umschalten vom Handy ohne OBS anzufassen. Komplett neue DB-Struktur, WebSocket live-switching, Scene-Switcher UI auf der Website.

⚡ Kern-Feature — ohne das ist alles andere nur ein statisches Overlay
2

🧩 Echte Widgets

Alert-Box mit Animationen wie StreamElements, Text-Widget, Timer (Countdown/Countup), Viewer-Zähler, Sub-Zähler, Recent Events, Social-Banner, Web-Source (iframe), Kamera-Placeholder.

📦 ~10 neue Widget-Typen
3

🎨 Vorlagen / Templates

Vorgefertigte Overlays (Gaming, Minimal, Purple Stream, Retro) die man als Startpunkt anwenden kann. Pro Szene eine passende Vorlage. Widgets werden automatisch angelegt.

🚀 Einfacher Einstieg für neue Streamer
4

📱 Fullscreen Editor

Hochkant + Querformat ohne speziellen Modus. Vollbild-Editor der den gesamten Bildschirm nutzt. Touch-Drag funktioniert, große Tap-Ziele, Scene-Switcher direkt im Editor.

📱 Macht alle anderen Features auf Handy nutzbar

Meine Empfehlung: Zuerst Szenen-System (1) + Fullscreen Editor (4) als Fundament, dann Widgets (2), dann Templates (3). So hat man schnell etwas das wirklich nutzbar ist.
